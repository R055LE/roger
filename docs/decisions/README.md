# Architecture Decision Records

Short, dated records of decisions that had real tradeoffs — the **why**, not the what (the code and
[`../../BACKLOG.md`](../../BACKLOG.md) have the what). One file per decision, numbered, append-only:
when a decision changes, add a new record that supersedes the old one rather than editing history.

Format is Nygard-lean — **Status / Context / Decision / Consequences**. Keep them short; an ADR that
needs a page is usually two decisions.

These first records were backfilled from the 2026-07 hardening/observability run — a good stress test
of the format. New decisions get the next number as they happen.
